59/** \brief plotter widget for scientific plots (uses JKQTBasePlotter to do the actual drawing)
60 * \ingroup jkqtpplotterclasses
61 *
62 * This class is a QWidget-wrapper around JKQTBasePlotter. It uses the tools from JKQTBasePlotter
63 * to display scientific plots. This class mostly adds the Widget for the output and adds different
64 * types of user interactions.
65 *
66 * <b>Please have a look at the documentation of JKQTBasePlotter for details on the management of graphs
67 * and the formating/styling of the plot and graphs!</b>
68 *
69 * The rest of this documentation ins split into sections that each treat a special topic, as outlines below:
70 *
71 * \tableofcontents
72 *
73 *
74 * \section JKQTPLOTTER_BASICUSAGE Basic Usage of JKQTPlotter
75 *
76 * JKQTPlotter is a plotter widget which wraps around a JKQTBasePlotter instanced that does the actual drawing.
77 * A basic usage of JKQTPlotter looks like this:
78 *
79 * \code{.cpp}
80 * // create a new JKQTPlotter instance
81 * JKQTPlotter* plot = new JKQTPlotter(parentWidget);
82 *
83 * // fill two vectors with dtaa for a graph:
84 * QVector<double> X, Y;
85 * fillDataVectors(X, Y);
86 *
87 * // make data available to the internal datastore of the plotter:
88 * size_t columnX=plot->getDatastore()->addCopiedColumn(X, "x");
89 * size_t columnY=plot->getDatastore()->addCopiedColumn(Y, "y");
90 *
91 * // create a graph/curve, which displays the data
92 * JKQTPXYLineGraph* graph1=new JKQTPXYLineGraph(plot);
93 * graph1->setXColumn(columnX);
94 * graph1->setYColumn(columnY);
95 * graph1->setTitle(QObject::tr("graph title"));
96 * plot->addGraph(graph1);
97 *
98 * // autoscale the plot
99 * plot->zoomToFit();
100 * // alternatively set the axis dimension by hand:
101 * plot->setXY(-10,10,-10,10);
102 * \endcode
103 *
104 * The result should look something like this:
105 *
106 * \image html jkqtplotter_simpletest1.png
107 *
108 * Starting from this basic example, you can observe several important principles:
109 * # Data is stored in an (internal) instance of JKQTPDatastore, which is accessible through
110 * JKQTPlotter::getDatastore().
111 * This datastore can either own its data (which is done here, as we copy the data into the store
112 * by calling JKQTPDatastore::addCopiedColumn(), or it can merely reference to the data (then
113 * data needs to be available as array of \c double values).
114 * # Naming conventions:
115 * - \b plot is the complete drawn image, including the axes, the graphs, the key and all other visual elements
116 * - <b>plot element</b> any sub element of the plot, e.g. a single coordinate axis, the key, but also any graph/curve
117 * - \b graph is a single curve/image/geometric element in the plot
118 * - <b>geometric element</b> is a special graph that does not represent a curve based on data from the JKQTPDatastore,
119 * but a single graphic element, like a rectangle/circle/line/..., some text, a single symbol
120 * - \b key is the legend of the plot
121 * - <b>coordinate axis</b> is each of the x- or y-axis (there might be addition axes, e.g. when showing a color-scale)
122 * .
123 * # Each graph is represented by a class derived from JKQTPPlotElement (in the example we instanciated a JKQTPXYLineGraph,
124 * which shows data as a scatter of symbols that may (or may not) be connected by a line).
125 * Creating the graph class does not yet add it to the plotter. To add it, call JKQTPlotter::addGraph(). Only
126 * after this sep, the graph is displayed. You can modify the apperance of the graph (e.g. colors,
127 * name in the key ...) by setting properties in the graph class instance.
128 * # You can auto-zoom the axis ranges of the plot by calling JKQTPlotter::zoomToFit(), or set them
129 * exlicitly by calling JKQTPlotter::setXY(). The user can later zoom in/out by the mouse (and other means).
130 * You can limit this zoom range by setting an absolute axis range, calling e.g. JKQTPlotter::setAbsoluteXY().
131 * The the user cannot zoom farther out than the given range(s).
132 * # If you want to style the plot itself, you need to set properties of the underlying JKQTBasePloter instance, which
133 * is accessible through JKQTPlotter::getPlotter(). If you want to style the coordinate axes, you can acces their
134 * representing objects by caling JKQTPlotter::getXAxis() or JKQTPlotter::getYAxis().
135 * .
136 *
137 * \see \ref JKQTPlotterSimpleTest and \see JKQTPlotterQtCreator

168 *
169 * JKQTPlotter offers a lot of user-interaction features out of the box. These are detailed below.
170 *
171 * \subsection JKQTPLOTTER_CONTEXTMENU Context Menu of JKQTPlotter
172 *
173 * The class JKQTPlotter has a context menu that already offers a lot of functionality.
174 *
175 * \image html jkqtplotter_defaultcontextmenu.png
176 *
177 * \image html contextmenu_graphvisibility.gif
178 *
179 * It allows to:
180 * <ul>
181 * <li> copy or save the data from the internal JKQTPDatastore</li>
182 * <li> copy or save the plot to an image file (PNG, PDF, ...), includes a softisticated export-preview dialog:
183 *
184 * \image html jkqtplotter_exportpreviewdialog.png
185 * </li>
186 * <li> print the plot (includes a softisticated print-preview dialog):
187 *
188 * \image html jkqtplotter_printpreview.png "Print Preview Dialog"
189 *
190 * \image html jkqtplotter_printpreview_relsize_mullinewidth_mulfonts.png "Print Preview Dialog with modified (relative) size, line-width and font-size"
191 * </li>
192 * <li> open a dialog with a table of the plot data:
193 *
194 * \image html jkqtplotter_datatabledialog.png "Data Table Dialog (with buttons to save or copy the data)"
195 * </li>
196 * <li> zoom into/out of the plot and determine an auto-zoom, which shows all of the plot data</li>
197 * <li> switch the visibility of the different graphs in the plot</li>
198 * </ul>
199 *
200 * You can activate this menu, by setting \c setContextMenuMode(jkqtpcmmStandardContextMenu); and binding a mouse-action (typically the right-click)
201 * to the context menu by calling \c deregisterMouseDragAction(Qt::RightButton,Qt::NoModifier) which deregisters any user actions from the rhs single
202 * mouse click. In that case, the context menu is opened by default (see setContextMenuMode() to choose the mode of the context menu).
203 *
204 *
205 * In addition JKQTPlotter provides several ways to customize this menu:
206 * - You can add additional actions to the JKQTPlotter, by calling `JKQTPlotter::addAction()` (i.e. using the default Qt mechanism)
207 * and/or by adding actions to the internal JKQTBasePlotter, using `JKQTBasePlotter::registerAdditionalAction()`
208 * - You can modify the menu, when it is displayed, by connecting a slot to the signal `JKQTPlotter::contextMenuOPened(x,y,menu)`:
209 * \code
210 * connect(plot, SIGNAL(contextMenuOpened(double, double, QMenu*)), this, SLOT(contextMenuOpened(double, double, QMenu*)));
211 *
212 * // ...
213 *
214 * void TestUserInteraction::contextMenuOpened(double x, double y, QMenu *contextMenu)
215 * {
216 * contextMenu->addSeparator();
217 * QAction* act=contextMenu->addMenu(QString("contextMenuOpened(x=%1, y=%2)").arg(x).arg(y))->addAction("user-added action");
218 * connect(act, &QAction::triggered, [x,y]() { QMessageBox::warning(nullptr, tr("Plot Context Menu"),
219 * tr("Context Menu was opened at x/y=%1/%2!").arg(x).arg(y),
220 * QMessageBox::Ok,
221 * QMessageBox::Ok); });
222 * labMouseAction->setText(QString("contextMenuOpened(x=%1, y=%2)").arg(x).arg(y));
223 * }
224 * \endcode
225 * .

256 * \subsection JKQTPLOTTER_ACTIONS QActions from a JKQTPlotter
257 * Often you want to allow the suer to operate a plot from a user-defined QToolBar or a QMenu/QMenuBar in your
258 * application (e.g. provide zooming commands ...). there are generally two ways to achieve this:
259 * <ol>
260 * <li> Simply connect home-brewn QAction instances to the slots provided by JKQTPlotter and JKQTBasePlotter.
261 * This also allows you to connect different plot properties to edit widgets in your forms.
262 * \code
263 * // add a checkbox to show and hide the position display label in the plot
264 * chkPositionDisplay=new QCheckBox(tr("show mouse cursor position"), this);
265 * chkPositionDisplay->setChecked(plot->isMousePositionShown());
266 * connect(chkPositionDisplay, SIGNAL(toggled(bool)), plot, SLOT(setMousePositionShown(bool)));
267 * \endcode
268 * </li>
269 * <li> For several functions (especially those also present in JKQTPlotter's context-emun, you can also find
270 * readily available QAction instances. these are available from JKQTBasePlotter (e.g. by JKQTBasePlotter::getActionPrint() ...).
271 * From JKQTPlotter you therefor have to use: <code>getPlotter()->getActionPrint()</code>
272 * \code
273 * // add some of the default QActions from the JKQTPlotter to the window menu
274 * // Some of the are also available in the context menu and toolbar of the JKQTPlotter
275 * // others are not
276 * QMenu* menuPlot=menuBar()->addMenu("Plot-Menu");
277 * menuPlot->addAction(plot->getPlotter()->getActionPrint());
278 * QMenu* menuPlotS=menuPlot->addMenu("Save ...");
279 * menuPlotS->addAction(plot->getPlotter()->getActionSaveData());
280 * menuPlotS->addAction(plot->getPlotter()->getActionSavePDF()); // not available from JKQTPlotter by default
281 * menuPlotS->addAction(plot->getPlotter()->getActionSavePlot());
282 * \endcode
283 * </li>
284 * </ol>
285 * \see \ref JKQTPlotterUserInteraction

289 * \subsection JKQTPLOTTER_USERMOUSEINTERACTION Mouse-Interaction in JKQTPlotter
290 *
291 * This section summarizes all user-interaction functions in JKQTPlotter that somehow relate to the mouse.
292 * These are:
293 * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEDRAG
294 * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSECLICK
295 * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEWHEEL
296 * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEMOVE
297 * .
298 *
299 * \note Zooming and Paning interactions apply to both axes when they are performed with the mouse
300 * inside the plot. They are limited to one of the two axes, when the mouse hovers over that axis
301 * (e.g. when you zoom by mouse-wheel while the mouse pointer is below the plot, over the x-axis,
302 * only the x-axis is affected by the operation).
303 *
304 * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEDRAG Actions When Dragging the Mouse
305 * JKQTPlotter offers several methods that allow to customize, how it reacts to mouse actions:
306 * - registerMouseDragAction() tells JKQTPlotter to perform a certain action (selected from JKQTPlotter::MouseActionMode)
307 * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
308 * By default JKQTPlotter calls these two registers in its constructors:
309 * \code
310 * registerMouseDragAction(Qt::LeftButton, Qt::NoModifier, JKQTPlotter::MouseActionMode::jkqtpmdaZoomByRectangle);
311 * registerMouseDragAction(Qt::LeftButton, Qt::ControlModifier, JKQTPlotter::MouseActionMode::jkqtpmdaPanPlotOnMove);
312 * \endcode
313 * Therefore by default you can draw a zoom rectangle with the left mouse button without modifiers
314 * and you can pan/drag the plot with the left mouse-button while keeping CTRL pressed.
315 * - deregisterMouseDragAction() deletes a previously defined user-interaction
316 * - clearAllRegisteredMouseDragActions() deletes all previously specified user-actions
317 * .
318 *
319 * Pressing the \c ESC key will stop the current JKQTPlotter::MouseActionMode.
320 *
321 * If e.g. the mode JKQTPlotter::MouseActionMode::jkqtpmdaZoomByRectangle is selected, while you drag the mouse, the
322 * zoom rectangle is drawn over the plot. You can modify the style of drawing using these functions:
323 * - setUserActionOverlayPen() sets the pen for (semi-transparent) overlay shapes, e.g. zoom rectangles
324 * - setUserActionOverlayBrush() sets the brush for (semi-transparent) overlay shapes, e.g. zoom rectangles
325 * - setUserActionOpaquePen() sets the pen for opaque overlay shapes, e.g. tool-tips
326 * - setUserActionOpaqueBrush() sets the brush for opaque overlay shapes, e.g. tool-tips
327 * - setUserActionMarkerPen() sets the pen for marker overlay shapes, e.g. with tool-tips
328 * - setUserActionMarkerBrush() sets the brush for marker overlay shapes, e.g. with tool-tips
329 * - setUserActionMarkerDiameter() sets the size of user-action markers
330 * - setUserActionMarkerType() sets the type of user-action markers
331 * .
332 *
333 * \image html zoomin_mouse_contextmenu.gif "Zooming with the mouse"
334 *
335 * \image html draw_rectangle.gif "Draw Rectangle User-Action"
336 *
337 * \image html drag_viewport.gif "Drag the Plot Viewport"
338 *
339 * \image html rulertool.gif "Measurement Ruler Tool"
340 *
341 * \image html tooltiptool.gif "Data Point Tooltip Tool"

344 * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSECLICK Actions After (Double-)Clicks on the Mouse Buttons
345 * The right mouse button has a special role: If it is single-clicked and no JKQTPlotter::MouseActionMode is specified
346 * for the vent, it opens the context menu, unless you call \c setContextMenuMode(jkqtpcmmNoContextMenu) .
347 * You can also use setContextMenuMode() to specify which type of context menu is shown. See JKQTPContextMenuModes
348 * for details on the available modes.
349 *
350 * For any mouse-click, one of the following signals is emitted:
351 * - plotMouseClicked() for any single-click (during the pressDown-Event!)
352 * - plotMouseDoubleClicked() for any double-click
353 * .
354 *
355 * The reaction to double-clicks of the mouse buttons can be specified separately. The possible
356 * actions are listed in JKQTPMouseDoubleClickActions. You can bind one of these actions
357 * to a specific click with these functions:
358 * - registerMouseDoubleClickAction() tells JKQTPlotter to perform a certain action (selected from JKQTPMouseDoubleClickActions)
359 * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
360 * By default JKQTPlotter calls this in its constructors:
361 * \code
362 * registerMouseDoubleClickAction(Qt::LeftButton, Qt::NoModifier, JKQTPMouseDoubleClickActions::jkqtpdcaClickMovesViewport);
363 * \endcode
364 * Therefore by default you can pan the plot/move the viewport by double clicking a location
365 * - deregisterMouseDoubleClickAction() deletes a previously defined user-interaction
366 * - clearAllRegisteredMouseDoubleClickAction() deletes all previously specified user-actions
367 * .
368 * The button to react to is specified as a parameter.
369 *
370 * \image html contextmenu_graphvisibility.gif
371 *
372 * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEWHEEL Actions When a Mouse Wheel Event Occurs
373 * The actions to be performed when the mouse hweel is operated are specified in JKQTPMouseWheelActions.
374 * You can bind one of these actions to the mouse-wheel (under the condition that a specified Qt::KeyboardModifiers
375 * is pressed) by using these functions:
376 * - registerMouseWheelAction() tells JKQTPlotter to perform a certain action (selected from JKQTPMouseWheelActions)
377 * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
378 * By default JKQTPlotter calls this in its constructors:
379 * \code
380 * registerMouseWheelAction(Qt::NoModifier, JKQTPMouseWheelActions::jkqtpmwaZoomByWheel);
381 * \endcode
382 * Therefore by default you can zoom into and out of the plot, using the mouse wheel.
383 * - deregisterMouseWheelAction() deletes a previously defined user-interaction
384 * - clearAllMouseWheelActions() deletes all previously specified user-actions
385 * .
386 * In addition the signal void plotMouseWheelOperated() is emitted whenever a mouse-wheel event occurs.
387 *
388 *
389 * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEMOVE Signaling When Mouse Moves
390 * In addition the signal plotMouseMove() is called whenever the mouse moves over the plot.
391 * Additional signals may be emitted, depending on the currently active JKQTPlotter::MouseActionMode.
392 *
393 * Also the current mouse position is shown above the graph by default (can be switched on or off
394 * using setMousePositionShown() ).
395 *
396 * \image html mousepositiondisplay.gif
397 *

399 * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_FASTRESIZING Fast Resizing of a Plot Widget

401 * As this is very timeconsuming, JKQTPlotter uses a different approach: Actual Redrawing is delayed a few 100ms
402 * and this delay is reset whenever the widget size changes. Only after the last delay expires, the plot is redrawn.
403 * In between an image of the last state of the plot is simple stretched/compressed to fill the new/current
404 * widget size. In sum this looks like this:
405 *
